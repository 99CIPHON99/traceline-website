Using Traceline

AI Features

Traceline's AI features work with any OpenAI-compatible endpoint — including a fully local, free, private model via LM Studio. Nothing is sent anywhere unless you point it at a cloud provider yourself.

Configuring a provider

Settings → AI Provider. Two supported shapes:

ProviderSetup
Local (LM Studio)Default. Point Base URL at http://localhost:1234/v1, load any model in LM Studio, no API key needed.
OpenAI-compatible cloudSet Base URL to your provider's endpoint (e.g. https://api.openai.com/v1) and paste an API key — stored encrypted at rest.

Use Test Connection after saving to confirm the endpoint actually responds before relying on it.

Generate Test Cases

Test Cases → AI Generate. Describe a feature in plain language (a couple of sentences minimum), pick a target suite, and review the drafted cases before accepting — nothing is written to the database until you explicitly accept. Accepted cases land in an "AI Generated" section within the chosen suite, tagged ai-generated.

You can also generate cases directly from a Requirement (Requirements → AI Cases) — the generated cases are automatically linked back to that requirement for traceability.

AI Run Summary

From a Test Run's toolbar, AI Summary produces an executive-style summary of the run's results — useful for a status update without reading every individual result.

Suggest Coverage

Analyzes a suite's existing test cases and suggests gaps — areas that look under-tested relative to the feature surface implied by the existing cases.

Cluster Failures

From a run with failures, Cluster Failures groups failing tests that likely share one root cause, with a suggested root cause and recommendation per cluster — then lets you draft one issue per cluster instead of filing duplicates for what's really one underlying bug.

Draft Issue

Any failed/blocked test result can draft a bug report from its steps, expected result, and your comment. If no AI provider is reachable, this still works — it falls back to a mechanical draft built directly from the test case data, so the button is never dead.

All AI endpoints are rate-limited (20 requests per 5 minutes per user) to prevent runaway usage against a paid API key.
