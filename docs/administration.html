Administration

Users, Settings & Licensing

Everything an admin needs: managing users, the Settings tabs, backups, and activating a Supporter license.

Users & roles

Admin → Users. Create, edit, deactivate, or delete users, and assign one of the five roles (see Core Concepts for what each can do). Deactivating a user frees up a Community Edition seat without deleting their history.

Changing your own password requires re-entering your current password first — this stops someone at an unlocked, already-logged-in machine from silently taking over the account.

Backups

cli/backup.php dumps the database (gzipped) and archives uploads/, keeping the 14 most recent of each. It also rotates any log file over 10 MB. Schedule it daily:

Register-ScheduledTask -TaskName "Traceline Backup" `
  -Action (New-ScheduledTaskAction -Execute "php.exe" -Argument "C:\path\to\cli\backup.php") `
  -Trigger (New-ScheduledTaskTrigger -Daily -At 2am)

Restore with:

gzip -dc backups/db-<stamp>.sql.gz | mysql -u root traceline

Licensing

Traceline's Community Edition supports up to 5 active users, free forever. A one-time Supporter donation unlocks unlimited users on that install — see the Support page for details on the donation itself.

Activating a license

  1. Make a Supporter donation and provide your email
  2. A signed license key arrives by email, in the form TL1.xxxxx.xxxxx
  3. Settings → License → paste the key → Activate
  4. Verified locally against a cryptographic signature — no external call required for this step

How verification works

License keys are signed with ECDSA (P-256/SHA-256). Traceline ships only the public half of the signing key — activation is verified entirely offline, so it works even on an air-gapped install. There's no license server your install has to reach just to keep running.

Traceline's code is fully open source under AGPL-3.0 regardless of edition — this only affects the official build's user-count convenience limit, not the code itself.

Revocation

In the rare fraud/chargeback case, a license can be revoked from the license-issuing side. Because verification is offline by design, revocation isn't instant — installs check a public revocation list opportunistically (roughly weekly, with a short timeout, and fail open if unreachable) rather than phoning home on every launch.
