Using Traceline

Test Execution

Test Runs are where test cases actually get executed and tracked. This page covers creating a run, the execution screen, and its keyboard shortcuts.

Creating a run

From a Project's Runs tab, click New Run, give it a name, and select which test cases to include — an entire suite, specific sections, or a hand-picked set. Traceline snapshots those cases into the run with an initial status of Untested.

The execution screen

Open a run and click Execute. Results are grouped by section, each showing the case title, priority, tags, preconditions, and (collapsed by default) its steps.

Keyboard shortcuts

The execution screen is built to run entirely from the keyboard once you're in a rhythm:

KeyAction
J / KNext / previous test
PMark Passed
FMark Failed
BMark Blocked
RMark Retest
SMark Skipped
CAdd a comment
EExpand steps

Bulk updates

Click Select to reveal checkboxes, select multiple results, then apply one status to all of them at once — useful for marking an obviously-unaffected group as Passed after a targeted fix.

Progress summary

The top of the run shows a live-updating breakdown (Passed / Failed / Blocked / Retest / Skipped / Untested) with a stacked progress bar — it updates immediately as you work through the run, no refresh needed.

Drafting an issue from a failure

Any failed or blocked result shows a bug icon — click it to draft a bug report pre-filled with the test case's steps, expected result, and your comment. If an AI provider is configured (see AI Features), the description is AI-written; otherwise it falls back to a mechanical draft built straight from the test case, so the button always works either way.

AI run summary & failure clustering

From the run's toolbar: AI Summary generates an executive summary of the run's results; Cluster Failures groups failures that likely share a root cause and lets you draft one issue per cluster instead of one per failing test. Both require an AI provider configured in Settings — see AI Features.
