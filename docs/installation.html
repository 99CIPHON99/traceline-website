Getting Started

Installation

Traceline is plain PHP + MySQL — no build step, no Node runtime required to run the app itself. This page covers a typical XAMPP install; see the repo's SETUP.md for the authoritative, always-current version.

Requirements

  • PHP 8.1+ with the pdo_mysql, openssl, and curl extensions (all enabled by default in XAMPP)
  • MySQL or MariaDB
  • Apache with mod_rewrite enabled

1. Get the files

Download or clone the repository, then copy the folder into your web root:

C:\xampp\htdocs\traceline\

2. Enable mod_rewrite

In XAMPP Control Panel → Apache → Config → httpd.conf, make sure this line has no leading #:

LoadModule rewrite_module modules/mod_rewrite.so

And that your <Directory> block allows overrides:

<Directory "C:/xampp/htdocs">
    AllowOverride All
    Require all granted
</Directory>

Restart Apache after this change.

3. Create the database

Easiest via phpMyAdmin: Import → choose install.sql → Go. Or from the command line:

mysql -u root -e "source C:/xampp/htdocs/traceline/install.sql"

4. Configure your environment

Copy the template and fill in real values — never ship the placeholder secrets beyond a local test install:

copy .env.example .env

Generate two real secrets:

php -r "echo bin2hex(random_bytes(32));"

Run that twice, and paste the results in as ENCRYPTION_KEY and INTERNAL_WORKER_SECRET. Leave DB_USER/DB_PASS/DB_NAME as the XAMPP defaults unless you changed something in Step 3.

.env holds real secrets. It's already gitignored and blocked from web access by .htaccess — never place it somewhere web-servable or commit it.

5. Log in

Visit http://localhost/traceline and sign in with:

Emailadmin@testtrack.local
PasswordAdmin1234!

Change this password immediately via Profile — it's a known default and this app is designed to be self-hosted on the open internet by anyone, including people who won't read this line.

Applying updates

Copy new files over your existing install, then apply any pending schema changes:

php cli/migrate.php status   # see what's pending
php cli/migrate.php          # apply it

Backups

Run manually or on a schedule (Task Scheduler / cron):

php cli/backup.php

Dumps the database (gzipped) and archives uploads/ into backups/, keeping the 14 most recent of each by default. Also rotates any log file over 10 MB. Copy the backups/ folder somewhere off-box — backups on the same disk as the app only protect against mistakes, not disk failure.

Going to production (HTTPS)

Flip these in .env once you have a real TLS certificate:

  • APP_ENV=production — hides PHP errors from responses
  • APP_URL=https://your-domain/traceline
  • SESSION_SECURE=true — only after HTTPS is actually working, or login will silently fail
  • Fresh ENCRYPTION_KEY / INTERNAL_WORKER_SECRET — never reuse values from a dev install
